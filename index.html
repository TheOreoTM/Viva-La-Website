Badges in red are required parameters, badges in gray are optional parameters and badges in green are filled-in parameters (most commonly seen used in subcommand groups)

# Example of required parameters

  • Member
  • Duration
  • Reason

# Example of optional parameters

  • Reason
  • Channel
  • Nickname

# Example of filled-in parameters

  • Promoting NFTs (reason)
  • #general (channel mention)
  • Inappropriate Nickname (nickname)

# Badges

Below are the list of badges used in the docs and what they mean. (Color variations not included)

Badge Meaning
Member A member in the server (@mention / ID)
User A user (usually means the user is not in the guild)
Duration A duration in the format 1m, 5h, 7d
Reason A reason for an action
Channel A channel in the server (#mention / ID)
Nickname A new nickname for a member (max length of 32)
Role A role in the server (@mention / ID)
Case ID A case ID (found in modlogs)
Bot A reference to Gir-Unit
Note A note for a member
Warning ID A warn Id found in warnings
Server The server
Query A query of a resource
Author The member who ran the command
XYZ A string
Subcommand Group A subcommand group
Subcommand A subcommand
Amount An amount
Hex Code A hex code
